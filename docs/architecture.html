🗃 数据库 Schema

model User {
  id        String   @id @default(cuid())
  slug      String   @unique          // URL 标识,如 "tourist"

  // 个人资料 (原 Profile 单例已并入)
  avatar    String   @default("/uploads/avatars/default.svg")
  name      String   @default("anonymous")
  bio       String   @default("")
  signature String   @default("")
  location  String   @default("")

  // 三平台 handle
  cfHandle  String   @default("")
  atcHandle String   @default("")
  ncHandle  String   @default("")

  // 三平台 rating 快照 (实时 + 历史最高)
  cfRating, cfRank, cfMaxRating, cfMaxRank
  atcRating, atcRank, atcMaxRating, atcMaxRank
  ncRating, ncRank, ncMaxRating, ncMaxRank

  createdAt, updatedAt
  ratingPoints RatingPoint[]     // 一对多
  submissions  Submission[]      // 一对多
}

model RatingPoint {
  id, userId, platform, date, rating
  @@unique([userId, platform, date])  // 同用户同平台同日期唯一
}

model Submission {
  id, userId, platform, problemId, problemName, problemUrl
  tags[], score, submitTime, language, verdict
  @@unique([userId, platform, problemId, submitTime])
}

model Contest {
  id, platform, name, date, time, duration, url
  @@unique([platform, name, date])
}

🔌 API 设计

读取类

端点逻辑
/api/usersPrisma findMany → 列表
/api/profilegetUserBySlug → 资料
/api/user查 freshness → 即时返回旧值 + 后台 refresh
/api/submissions同上，everPopulated 检测首次同步拉取
/api/contestsPrisma findMany → 赛程

写入类

端点逻辑
/api/profile (POST)upsert User → 写资料/handle
/api/upload-avatarmulter 解析 → 存 public/uploads/
/api/cron/refresh遍历所有 User → refreshUserRating + refreshUserSubmissions

Cron 刷新流程

外部 cron (Vercel / GH Actions)
  ⬇  GET /api/cron/refresh  (Bearer CRON_SECRET)
  ⬇  遍历所有 User (并发度 3, 避免 CF 限流)
  ⬇  refreshUserRating()   → CF / AtC / NC API → Prisma upsert
  ⬇  refreshUserSubmissions() → CF / AtC / NC API → Prisma createMany(skipDuplicates)
  ⬇  返回 { updated, partial, failed, errors }

🔄 核心数据流

首次访问新用户

浏览器 → GET /u/tourist
  ⬇  RSC: getUserBySlug("tourist") → Prisma → Neon
  ⬇  <HomePageClient profile={...} />
  ⬇  CSR: useEffect → refreshAll()
  ⬇  并行 fetch:
  │   ├ /api/user?slug=tourist      → everPopulated=false → 同步拉取 CF/AtC/NC → 返回真数据
  │   ├ /api/submissions?slug=tourist → 同上, 首次同步
  │   └ /api/contests                → 缓存命中直接返回
  ⬇  setState → 渲染 rating + 热力图 + 日志

日常访问 (数据新鲜)

浏览器 → GET /u/tourist
  ⬇  CSR: localStorage 缓存 → 秒渲染
  ⬇  fetch /api/user → (now - updatedAt) < 4h → 即时返回旧值
  ⬇  后台 refresh 异步更新 → 下次访问拿新数据

Cron 定时刷新 (每 4 小时)

Vercel Cron / GH Actions
  ⬇  GET /api/cron/refresh (Bearer CRON_SECRET)
  ⬇  prisma.user.findMany()
  ⬇  并发度 3, 逐个 user:
  │   ├ refreshUserRating()      → CF/AtC/NC API → User + RatingPoint
  │   └ refreshUserSubmissions() → CF/AtC/NC API → Submission
  ⬇  allSettled 降级: 单平台失败不影响其他
  ⬇  返回 { updated, partial, failed, errors }

localStorage 缓存策略

Key 格式:  cp-blog:<slug>:<suffix>
  cp-blog:tourist:rating     → { cf: RatingData, atc: RatingData, nc: RatingData }
  cp-blog:tourist:subs       → { cf: SubmissionDay[], atc: ..., nc: ... }
  cp-blog:tourist:section    → "home" | "submissions" | "contests"

版本管理:  cp-blog-cache-version = "v5"
  版本不匹配 → 清掉所有旧缓存 → 重新拉取
